6. PRICE ACCURACY AND STALENESS RULES
==================================================

Every price shown in the app should carry metadata:

{
  symbol,
  assetClass,
  price,
  change,
  changePercent,
  provider,
  timestamp,
  isLive,
  isDelayed,
  isStale,
  isFallback,
  isDemo,
  marketSession,
  confidence,
  error
}

Freshness rules:
- Equities during market hours: stale if older than 60–120 seconds unless delayed feed is expected.
- Equities after hours: label as after-hours/last close/extended depending source.
- Crypto: stale if older than 30–90 seconds for live display.
- Fundamentals: can be older but must show reporting period/date.
- News/research: show published/source timestamp.

If data is stale:
- show “Stale”
- show last updated time
- do not show green “LIVE”
- optionally dim the value
- do not use stale data for new alerts unless allowed by settings

If provider fails:
- try next fallback
- label fallback provider
- show warning in Settings/source health
- do not silently hide failure
