SECURITY REQUIREMENTS
=====================

Important:
API keys must be handled securely.

Do not put secret keys directly in frontend JavaScript.
Do not expose raw keys in client responses.
Do not log full API keys.
Do not store full keys in localStorage.

Preferred implementation:

* Store API keys server-side.
* Use environment variables or secure server-side storage.
* If persistent user-entered keys require database storage, encrypt them server-side before saving.
* Only return masked key state to frontend, such as:
  “••••••••••••ABCD”
* Show whether a key exists, but never reveal the full key after save.
* The user can replace or delete the key.
* Add clear comments explaining where keys are stored.

If true secure server-side persistent key storage is not available in this Replit setup:

* Add the UI and backend structure anyway.
* Add clear TODO comments.
* Do not fake saving.
* Prefer using Replit Secrets/environment variables.
* Show instructions in the UI: “Add this key in Replit Secrets for production safety.”
